# 元圭与 VCC 1.1.2

## 1.1.2 翻译体验与性能升级

- 重构统一翻译管线：截图 OCR 段落支持批量翻译、请求合并、任务取消和内存缓存；默认继续使用免费的 YuanGUI 系统快捷指令，不自动调用付费在线 AI。
- 截图翻译改为结构化 OCR 与语义句子回填。系统快捷指令使用内部序号保持句子与原位置的对应关系，并在显示前强制清理序号，避免译文错位或内部标记泄漏。
- 改进网页、邮件、列表、多列和包含 URL 的复杂截图识别；只有整行都是 URL、邮箱或纯数字时才跳过翻译，包含链接的正常句子仍会翻译。
- 优化截图覆盖排版：译文保持原文横向锚点，只在同列安全空白内扩展；根据句子长度与空间自动选择字号，最低 7pt，支持自动换行并避免省略号和相邻文本遮挡。
- 截图翻译工具条新增放大与缩小，保留中英对照、复制、拖动和 Esc 一键关闭；翻译层保持在触发时的桌面空间。
- 增强网页与其他应用的划词读取兼容性，并在没有选中文字时继续支持手动输入翻译。
- 修复划词翻译窗口按较大字体估算译文高度，导致译文下方出现大块空白的问题。
- 增加截图布局夹具、随机布局属性测试和离线性能测试，覆盖错位、重叠、省略、内部标记和窗口缩放等回归场景。

# 元圭与 VCC 1.1.1

## 1.1.1 稳定性修复

- 修复发布版在其他电脑打开“快捷工具”时因 SwiftPM 资源路径失效而闪退的问题，并避免访问开发电脑上的构建目录。

以下为 1.1 版本原有更新内容：

## 截图与翻译

- 新增区域截图和图片编辑，可添加画笔、文字、箭头、形状、高亮与马赛克，并保存到文件夹或复制到剪贴板。
- 新增截图 OCR 翻译、划词翻译与手动输入翻译；支持编辑原文后重新翻译、快捷复制译文，并在可编辑位置替换原文。
- 默认快捷键调整为：`Control-A` 区域截图、`Control-Shift-A` 截图翻译、`Control-Z` 划词翻译，均可在设置中修改。
- 系统翻译通过 YuanGUI 快捷指令调用 Apple 翻译，无需付费 API Key；仍可选用系统离线模型或在线 AI。

## AI 与快捷工具

- AI 对话支持 SSE 流式回复，单次最大输出提升至 4096 tokens，并完善设置页图标与生成状态显示。
- 状态栏新增“工具”页，集中提供 AI 对话、清理屋、软件卸载、截图、翻译、废纸篓和设置入口。
- 翻译窗口加入“元圭与 VCC 翻译小屋”标题，并调整为更紧凑清晰的布局。

## 音乐与视觉修复

- 歌词匹配支持歌曲名与歌手字段互换，兼容资料库中信息位置颠倒的歌曲。
- 修复手动切换 Bilibili 歌曲后，播放列表选中框仍停留在旧歌曲的问题。
- 优化元圭待机与聊天动作素材的透明边缘，减少腿部等位置的白边。

支持 macOS 15 或更高版本。此 DMG 为个人分享版，使用临时签名。
