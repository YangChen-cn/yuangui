# 元圭与 VCC 1.0.5

本版本在保留 1.0.4 字幕、歌词缓存、歌曲信息编辑和深色模式改进的基础上，重点优化播放器交互、同步性能和更新体验。

## 当前版本重要更新

- 完整播放器歌词扩展为七行窗口，支持鼠标点击选中后滚动预览、平滑跟随和点击歌词跳转；第一次点击只选中歌词界面，鼠标离开后自动恢复播放器滚动。
- 歌词加载期间区分“正在加载歌词”“暂无同步歌词”和“播放歌曲后显示歌词”，长歌词窗口保持稳定布局。
- 分离浏览来源与实际播放来源。切换 Apple Music 与 Bilibili 页面不会意外暂停另一播放器；只有主动播放时才切换实际播放源。
- 导入 Bilibili URL 后不再自动播放，改为显示“开始播放 / 继续浏览”，提示会在短暂展示后自动隐藏。
- 新增随机播放模式，保留顺序播放、列表循环和单曲循环。
- Apple Music 同步改为后台串行 worker：每 2.5 秒读取一次真实状态，播放期间使用本地时钟平滑推进，封面只在曲目变化时异步加载，避免 AppleScript 阻塞主线程。
- 更新检查界面重新排版 GitHub Release，长标题、Markdown 列表、段落和错误信息都可以正常换行。

## 延续的 1.0.4 重要更新

- 修复 Bilibili 播放器接口偶发保留正确 BV/CID、却返回其他视频字幕轨的问题；AI 字幕校验 `aid` 与 `cid`，人工字幕通过重复轨道 ID 确认。
- 已缓存的错误 Bilibili 字幕会在解析到正确字幕地址后自动失效并重新下载。
- Apple Music 的 LRCLIB 歌词使用稳定缓存键，忽略时长微小变化、大小写及空格差异，重启后无需重复匹配。
- Bilibili 登录入口显示昵称与 UID；歌曲名和歌手可以单独修改，不触发 LRCLIB，也不会删除现有歌词。
- 歌词偏移支持直接输入秒数和 0.1 秒步进微调，并统一状态栏、完整播放器和歌词设置弹窗的操作方式。
- 提高 AI 回复与日常聊天气泡的不透明度，并使用自适应背景色改善深色模式和深色桌面的文字对比度。
